## 2026-07-22 - DuckDB and NumPy Rolling Operations Optimization
**Learning:** In DuckDB, looping over `fetchall()` for bulk data extraction is a significant bottleneck. Using `fetchnumpy()` alongside `np.where(arr[:-1] != arr[1:])[0] + 1` to find boundaries and `np.split` handles bulk grouping entirely in C/C++ bypassing slow Python-level dictionary loops.
For rolling array operations (e.g. `bollinger`'s mean/std and `cmf`'s sum), Python loops over array slices carry huge overhead. `np.lib.stride_tricks.sliding_window_view` provides a robust ~30-100x speedup while strictly maintaining purely numerical arrays. Also, `np.divide` with the `where` parameter is crucial to maintain pure vectorized division securely without throwing division-by-zero warnings or polluting with NaNs.
**Action:** When extracting grouped time-series data from DuckDB, prioritize `fetchnumpy` + `np.split`. When performing rolling math across Numpy arrays, always use `sliding_window_view` instead of Python `for` loops.

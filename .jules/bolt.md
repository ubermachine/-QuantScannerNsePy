## 2025-02-18 - DuckDB `fetchnumpy()` overhead in Python bulk loading
**Learning:** Using `fetchall()` with a large DuckDB query returning hundreds of thousands of rows causes severe overhead during Python object instantiation and subsequent loop-based array construction.
**Action:** When performing bulk loading across multiple groupings (like multiple tickers), use `fetchnumpy()` directly and then segment the resulting large flat arrays using vectorized boundary detection (`np.where` on string column changes) and `np.split`. This provides substantial speedups compared to appending arrays row by row in a standard `for` loop. Additionally, remember to convert `numpy.datetime64` arrays back to Python `datetime` objects via `pandas.to_datetime().to_pydatetime()` for downstream compatibility.
